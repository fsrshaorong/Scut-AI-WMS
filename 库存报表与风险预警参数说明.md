## 1. 概述

库存报表模块（`/api/stock/report`）对每笔库存记录进行 **四级动态评级**，帮助仓储管理人员快速识别异常物料。评级结果基于三种维度：

| 维度 | 判定内容 | 依赖数据 |
|:---|:---|:---|
| **时间维度** | 物料是否长期闲置 | 最后出库日期（`outbound_histories` 表） |
| **供应安全维度** | 库存是否低于补货安全线 | 日均消耗、补货提前期、安全库存 |
| **资金效率维度** | 库存是否过度积压 | 日均消耗、高储控制天数 |

---

## 2. 四级评级体系

评级按优先级从高到低判定，一旦命中即返回，不再继续判断后续级别：

```
库存 = 0 ？
  ├─ 是 → 低储 (LOW_STOCK)
  └─ 否 → 闲置 ≥ 90 天 ？
              ├─ 是 → 呆滞 (DEAD_STOCK)
              └─ 否 → 库存 < 补货预警线 ？
                          ├─ 是 → 低储 (LOW_STOCK)
                          └─ 否 → DOHF > 高储上限 ？
                                      ├─ 是 → 高储 (HIGH)
                                      └─ 否 → 正常 (NORMAL)
```

### 2.1 呆滞 (DEAD_STOCK) — 优先级最高

**定义**：库存超过 90 天未发生任何出库行为，无论库存量多少。

**判定条件**：
- 库存量 > 0
- 最后出库日期距今 ≥ 90 天

**数据来源**：
- `outbound_histories` 表：按 `material_code` 查询 `MAX(created_at)` 获取最后出库时间
- 若物料从未出库，则以最早入库二维码的 `created_at` 为参考起点

**业务含义**：长期不流动的库存占用仓储空间和资金，且存在过期、过时、报废风险。即使库存量在"正常"范围内，只要不流动就是问题。

**行业参考**：多数制造企业将 90 天无出库定义为"滞料"；快消品行业可能缩短至 30-60 天；重型装备行业可能放宽至 180 天。

### 2.2 低储 (LOW_STOCK)

**定义**：当前库存量低于补货预警线，存在断供风险。

**判定条件**：
```
当前库存 < (日均销量 × 补货提前期) + 安全库存
```

**业务含义**：按当前消耗速度，在补货所需的时间内库存将被耗尽。需要立即发起采购或调拨。

### 2.3 高储 (HIGH)

**定义**：库存持有天数（DOHF）远超控制上限，存在资金浪费和呆滞风险。

**判定条件**：
```
DOHF = 当前库存 ÷ 日均销量
DOHF > maxStockDays（高储控制天数上限）
```

**业务含义**：现有库存即使按正常速度消耗，也需要异常长的时间才能消化。建议暂缓采购、优先消耗。

### 2.4 正常 (NORMAL)

**定义**：库存处于健康区间，既不缺货也不积压。

**判定条件**：以上三条均不满足。

## 3. 核心计算公式

### 3.1 日均消耗量

```
日均消耗 = Σ(近30天出库流水扣减量) ÷ 30
```

- 查询 `outbound_histories` 表中 `created_at >= NOW() - 30天` 的记录
- 汇总 `deduct_qty` 字段
- 若无出库记录，返回 `0.0`（而非硬编码默认值）

### 3.2 补货预警线

```
预警线 = (日均销量 × 补货提前期天数) + 安全库存量
```

| 参数 | 来源 | 说明 |
|:---|:---|:---|
| `日均销量` | `computeDailyConsume()` | 近 30 天实际出库平均 |
| `补货提前期` | `inventories.lead_time_days` | 下单→到货→上架全周期（默认 7 天） |
| `安全库存` | `inventories.safety_stock` | 吸收需求波动和供应不确定性的缓冲量（默认 0） |

> **为什么包含安全库存**：即使补货准时到达，需求波动也可能在提前期内耗尽库存。安全库存为这种不确定性提供缓冲。

### 3.3 DOHF（库存未来持有天数）

```
DOHF = 当前库存量 ÷ 日均销量
```

- 若日均销量为 0，DOHF 记为极大值（9999），代表"几乎无限期持有"
- 若库存为 0，DOHF 记为 0

> DOHF（Days of Inventory On Hand - Future）是衡量库存可维持天数的核心 KPI。与传统的"库存周转天数"不同，DOHF 使用**未来预测需求**而非历史消耗，但在当前实现中默认为近 30 天日均消耗。

### 3.4 闲置天数

```
闲置天数 = NOW() - 最后出库日期
```

- 若从未出库：`闲置天数 = NOW() - 最早入库二维码日期`
- 若入出库记录均无：`闲置天数 = 0`（视为新物料）

## 4. 数据库参数说明

`inventories` 表中每种物料可独立配置以下参数：

| 字段 | 类型 | 默认值 | 说明 |
|:---|:---|:---|:---|
| `stock_qty` | INT | 0 | 当前仓库实物库存现存量（件），由出入库操作自动更新 |
| `min_stock_days` | INT | 3 | 低储参考天数，用于前端显示和 AI 模型参考 |
| `max_stock_days` | INT | 15 | 高储控制天数上限，DOHF 超过此值判定为高储 |
| `safety_stock` | INT | 0 | 安全库存量（件），补货预警线的组成部分 |
| `lead_time_days` | INT | 7 | 补货提前期天数（下单→供应商生产→物流→质检→上架） |

### 参数配置示例

```
物料 A（大灯总成）：日耗 10 件，供应商 5 天到货，预留 50 件安全库存
  → lead_time_days=5, safety_stock=50
  → 预警线 = 10×5+50 = 100 件
  → 当前库存 200 件 → DOHF=20 天，若 max_stock_days=25 → 正常

物料 B（ECU）：日耗 10 件，供应商 7 天到货，预留 50 件安全库存
  → lead_time_days=7, safety_stock=50
  → 预警线 = 10×7+50 = 120 件
  → 当前库存 50 件 → 50 < 120 → 低储！
```

## 5. API 响应字段

`GET /api/stock/report` 返回 `StockReportVO` 数组：

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `materialCode` | String | 物料号 |
| `materialName` | String | 物料名称 |
| `stockQty` | Integer | 当前库存量（件） |
| `minStockDays` | Integer | 低储控制天数（参考值） |
| `maxStockDays` | Integer | 高储控制天数上限 |
| `safetyStock` | Integer | 安全库存量（件） |
| `leadTimeDays` | Integer | 补货提前期（天） |
| `dailyConsume` | Double | 近 30 天日均消耗（件/天） |
| `dohf` | Double | 库存未来持有天数 |
| `lastOutboundDate` | DateTime | 最后一次出库时间 |
| `idleDays` | Integer | 闲置天数（自最后出库至今） |
| `ruleEvaluation` | String | 评级结果：`LOW_STOCK` / `DEAD_STOCK` / `HIGH` / `NORMAL` |
| `updatedAt` | DateTime | 库存最后更新时间 |

### 筛选参数

| 参数 | 说明 |
|:---|:---|
| `materialCode` | 物料号模糊检索 |
| `alarmStatus` | 按评级筛选：`NORMAL` / `LOW` / `HIGH` / `DEAD_STOCK` |

> 注意：`alarmStatus=LOW` 匹配 `ruleEvaluation=LOW_STOCK`，前端筛选下拉框使用 `LOW` 以简化交互。

## 7. 配置调优指南

### 7.1 呆滞阈值调整

呆滞判定阈值 `DEAD_STOCK_DAYS = 90` 定义在 `StockServiceImpl.java` 常量中：
```java
private static final int DEAD_STOCK_DAYS = 90;
```
根据行业特性调整：
- 快消品 / 食品：30-45 天
- 汽车零部件（当前）：90 天
- 重型装备 / MRO：180 天

### 7.2 补货提前期设置

`lead_time_days` 应包含完整的补货周期：
```
提前期 = 订单处理 + 供应商生产 + 物流运输 + 质检入库 + 上架
```
典型值：国内供应商 3-7 天，进口件 30-60 天。

### 7.3 安全库存设置

安全库存用于吸收需求和供应的不确定性：
```
安全库存 = Z × σ × √提前期
```
- `Z`：服务水平系数（95% → 1.65）
- `σ`：需求标准差
- `√提前期`：提前期的平方根

简化实践中可直接设定为 `日均销量 × 3~7 天` 的经验值。

### 7.4 高储上限设置

`max_stock_days` 应小于物料保质期或产品生命周期：
- 保质期 180 天的物料：maxStockDays ≤ 120
- 换代频繁的电子料：maxStockDays ≤ 30
- 通用标准件：maxStockDays ≤ 60

## 8. 示例演算

以 **M_PART_005 刹车片组件** 为例（种子数据实际值）：

### 输入参数

| 参数 | 值 |
|:---|:---|
| 当前库存 | 40 件 |
| 近 30 天日均消耗 | 9.0 件/天 |
| 补货提前期 | 5 天 |
| 安全库存 | 50 件 |
| 高储上限 | 15 天 |
| 最后出库日期 | 3 天前 |

### 演算过程

```
1. 呆滞判定：闲置 3 天 < 90 天 → 不触发

2. 低储判定：
   预警线 = 9.0 × 5 + 50 = 95 件
   当前库存 40 < 95 → 低储！⚠️

3. 高储判定：（已命中低储，跳过）

最终评级：LOW_STOCK（低储）
```
